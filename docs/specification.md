## 1. 서비스 개요

### 1.1 서비스 소개

**ProtestAlert(집회알림)**는 대한민국 전역에서 예정된 집회 및 시위 일정을 사용자에게 제공하는 모바일 애플리케이션입니다.

사용자가 관심 있는 지역을 설정하면, 해당 지역의 집회 일정을 손쉽게 확인할 수 있습니다. 출퇴근이나 외출 계획 시 집회로 인한 교통 혼잡을 미리 파악하여 대비할 수 있도록 돕는
것이 핵심 가치입니다.

### 1.2 서비스 배경

- 대규모 집회는 도심 교통에 큰 영향을 미침
- 사전에 집회 정보를 알면 동선 계획에 도움이 됨
- 기존에는 집회 일정을 한눈에 확인하기 어려웠음
- 경찰청 집회신고 데이터를 기반으로 정확한 정보 제공

### 1.3 타겟 사용자

| 사용자 유형      | 설명                        | 핵심 니즈              |
|-------------|---------------------------|--------------------|
| **출퇴근 직장인** | 도심 출퇴근 시 교통 정보가 중요한 사용자   | 집회로 인한 교통 혼잡 사전 파악 |
| **운전자**     | 도심 운전이 잦은 사용자             | 우회 경로 사전 계획        |
| **외출 계획자**  | 주말 외출 시 혼잡 지역을 피하고 싶은 사용자 | 집회 일정에 따른 외출 시간 조정 |
| **지역 주민**   | 집회 빈도가 높은 지역 거주자          | 일상생활 영향 최소화        |

### 1.4 핵심 가치 제안 (Value Proposition)

```
"내 관심 지역의 집회 일정을 한눈에 파악하여
더 똑똑한 이동 계획을 세울 수 있습니다"
```

**주요 편익**:

- 집회로 인한 예상치 못한 교통 체증 방지
- 관심 지역만 필터링하여 필요한 정보만 확인
- 최신 데이터 자동 동기화로 정확한 정보 제공
- 오프라인에서도 이전에 받은 데이터 조회 가능

#### 3.1.3 온보딩 UI 요소

| 요소        | 설명                        |
|-----------|---------------------------|
| **안내 문구** | "관심 지역 선택 후 완료 버튼을 눌러주세요" |
| **지역 칩**  | 3열 그리드로 배치된 선택 가능한 지역 버튼  |
| **완료 버튼** | 최소 1개 이상 선택 시 활성화         |

#### 3.1.4 지역 칩 상태

| 상태      | 배경색             | 테두리              | 텍스트 색상          | 체크 아이콘 |
|---------|-----------------|------------------|-----------------|--------|

| **미선택** | 회색 (#F3F4F6)    | 없음               | 진한 회색 (#374151) | 없음     |

#### 3.1.5 온보딩 비즈니스 로직

| 조건           | 처리                             |
|--------------|--------------------------------|
| 선택된 지역 0개    | 완료 버튼 비활성화 (회색)                |
| 선택된 지역 1개 이상 | 완료 버튼 활성화 (파랑)                 |
| 완료 버튼 클릭     | 선택된 지역 저장, 온보딩 숨김 처리, 일정 목록 표시 |

#### 3.1.6 지역 목록

지역 목록은 **서버(Supabase)의 `regions` 테이블에서 동적으로 관리**됩니다.

- 지역 추가/삭제는 서버 데이터베이스에서 관리
- 앱은 서버에서 받아온 지역 목록을 그대로 표시
- 지역명 순으로 정렬하여 표시

> **참고**: 실제 지원 지역은 Supabase 관리자 페이지에서 확인해주세요.

#### 3.2.4 날짜 헤더 (Date Header)

각 날짜별 집회 목록의 상단에 표시되는 날짜 정보입니다.

**표시 형식**: `YYYY년 MM월 DD일 (요일)`

**예시**: `2025년 01월 18일 (토)`

**요일별 색상 처리**:

| 요일  | 텍스트 색상       | 비고    |
|-----|--------------|-------|
| 월~금 | 기본 색상 (검정)   | 평일    |
| 토요일 | 파랑 (#2B7FFF) | 주말 강조 |
| 일요일 | 빨강 (#E11D48) | 주말 강조 |

**오늘 표시**:

- 해당 날짜가 오늘인 경우 날짜 옆에 `[오늘]` 칩 표시
- 칩 배경색: 빨강 (#E11D48)
- 칩 텍스트: 흰색

#### 3.2.5 일정 카드 (Schedule Item)

개별 집회 정보를 카드 형태로 표시합니다.

**카드 레이아웃**:

```
┌─────────────────────────────────────────────────────┐
│ [시간 칩]                               [지역 칩]   │
│ 장소명                                              │
│ ─────────────────────────────────────────────────   │
│ 👥 참여자 N,NNN명                                   │
│ 🛡️ [관할1] [관할2] [관할3] ...                     │
└─────────────────────────────────────────────────────┘
```

**카드 구성 요소**:

| 요소         | 설명                 | 예시               |
|------------|--------------------|------------------|
| **시간 칩**   | 집회 시작~종료 시간        | `09:00 ~ 22:00`  |
| **지역 칩**   | 집회 지역              | `서울`             |
| **장소명**    | 집회 장소 상세           | `서울광장 및 세종대로 일대` |
| **참여자 수**  | 예상 참가 인원 (천 단위 콤마) | `👥 참여자 50,000명` |
| **관할 경찰서** | 담당 경찰서 목록          | `🛡️ [중부] [서대문]` |

**시간 표시 규칙**:

| 상황             | 표시              |
|----------------|-----------------|
| 시작/종료 시간 모두 있음 | `HH:MM ~ HH:MM` |
| 시작 시간만 있음      | `HH:MM ~ --:--` |
| 종료 시간만 있음      | `--:-- ~ HH:MM` |
| 둘 다 없음         | `--:-- ~ --:--` |

#### 3.2.6 데이터 필터링 규칙

| 규칙        | 설명                                 |
|-----------|------------------------------------|
| **지역 필터** | 사용자가 팔로우한 지역의 집회만 표시               |
| **날짜 필터** | 최근 7일 이내의 집회만 표시                   |
| **정렬 기준** | 날짜 내림차순 (최신순) → 같은 날짜 내 시작 시간 오름차순 |

#### 3.2.7 빈 상태 (Empty State)

관심 지역을 설정했지만 해당 기간 내 집회가 없는 경우:

- (현재 별도 빈 상태 UI 없음, 빈 리스트로 표시)

### 3.5 데이터 동기화

#### 3.5.1 동기화 개요

앱은 서버(Supabase)로부터 최신 집회 데이터를 받아 로컬에 저장합니다.

#### 3.5.2 동기화 시점

| 시점         | 동기화 수행                  |
|------------|-------------------------|
| **앱 시작 시** | 자동 동기화 (백그라운드)          |
| **백그라운드**  | WorkManager를 통한 주기적 동기화 |

#### 3.5.3 동기화 상태 표시

데이터 동기화 중일 때 화면 상단에 로딩 인디케이터(CircularProgressIndicator)가 표시됩니다.

**표시 조건**:

- 데이터 동기화 중
- 피드 데이터 로딩 중
- 온보딩 데이터 로딩 중

#### 3.5.4 오프라인 지원

| 상태       | 동작                                 |
|----------|------------------------------------|
| **온라인**  | 서버에서 최신 데이터 동기화, 로컬 DB 업데이트        |
| **오프라인** | 이전에 저장된 로컬 데이터 표시, 스낵바로 오프라인 상태 알림 |

**오프라인 알림 메시지**: "인터넷에 연결되어 있지 않습니다" (무기한 표시)

## 5. 사용자 시나리오

### 5.1 시나리오 1: 첫 사용자의 앱 시작

**상황**: 김서울 님이 앱을 처음 설치하고 실행합니다.

**플로우**:

1. 앱 아이콘 탭하여 실행
2. 스플래시 화면 표시 (약 1초)
3. 온보딩 화면 표시
    - 안내 문구: "관심 지역 선택 후 완료 버튼을 눌러주세요"
    - 17개 지역 칩이 3열로 표시됨
4. "서울"과 "경기" 지역 탭하여 선택
    - 선택된 지역은 파란 테두리 + 체크 아이콘으로 표시
    - 완료 버튼 활성화됨
5. "완료" 버튼 탭
6. 일정 화면으로 전환
    - 서울, 경기 지역의 집회 일정이 날짜별로 표시됨
7. 원하는 날짜의 집회 카드에서 시간, 장소, 참가자 수 확인

### 5.2 시나리오 2: 재방문 사용자의 일정 확인

**상황**: 박부산 님이 앱을 다시 열어 이번 주말 집회 일정을 확인합니다.

**플로우**:

1. 앱 실행
2. 스플래시 화면 후 바로 일정 화면 표시 (온보딩 생략)
3. 상단에 로딩 인디케이터 표시 (데이터 동기화 중)
4. 동기화 완료 후 최신 집회 일정 표시
5. 스크롤하여 주말(토, 일) 일정 확인
    - 토요일 날짜 헤더의 "(토)"가 파란색으로 표시
    - 일요일 날짜 헤더의 "(일)"이 빨간색으로 표시
6. 오늘 날짜의 집회는 [오늘] 칩으로 강조됨

### 5.3 시나리오 3: 관심 지역 변경

**상황**: 이대전 님이 대전에서 서울로 이사하여 관심 지역을 변경합니다.

**플로우**:

1. 앱 실행
2. 하단 탭에서 "설정" 탭 선택
3. 설정 화면에서 현재 선택된 "대전" 확인 (체크 표시됨)
4. "대전" 탭하여 선택 해제
5. "서울" 탭하여 선택
6. 변경 사항 자동 저장됨
7. "일정" 탭으로 이동
8. 서울 지역의 집회 일정만 표시됨

### 5.4 시나리오 4: 피드백 전송

**상황**: 최인천 님이 앱 사용 중 개선 사항을 발견하여 의견을 전달합니다.

**플로우**:

1. 설정 탭 이동
2. "문의 · 건의" 메뉴 탭
3. 피드백 모달 표시
4. 텍스트 영역에 의견 입력
    - "알림 기능이 있으면 좋겠습니다. 집회 당일 아침에 알림을 받고 싶어요."
5. 입력 시 "전송" 버튼 활성화됨 (파란색으로 변경)
6. "전송" 버튼 탭
7. 서버로 피드백 전송
8. 성공 시:
    - 체크 아이콘과 "소중한 의견 감사드립니다" 메시지 표시
    - 1.5초 후 모달 자동 닫힘
9. 설정 화면으로 복귀

### 5.5 시나리오 5: 오프라인 상태에서 앱 사용

**상황**: 지하철 이동 중 네트워크가 끊긴 상태에서 앱을 확인합니다.

**플로우**:

1. 앱 실행 (지하철 터널 구간, 오프라인)
2. 스플래시 화면 표시
3. 일정 화면 표시
4. 하단에 스낵바 표시: "인터넷에 연결되어 있지 않습니다"
5. 이전에 동기화된 집회 일정은 정상적으로 표시됨
6. 스크롤하여 저장된 일정 확인 가능
7. 터널 구간 통과 후 네트워크 복구
8. 스낵바 자동 사라짐
9. 백그라운드에서 자동 동기화 수행
